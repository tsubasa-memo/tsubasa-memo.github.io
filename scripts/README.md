# scripts/

このディレクトリはサイト保守用のスクリプト置き場。

## glossary-linker.py

本編記事の本文中にある専門用語を、用語集（`/glossary/*.html`）へインラインリンク化するツール。

### 使い方

```bash
# 差分プレビュー（実際の書き換えはしない）
python3 scripts/glossary-linker.py

# 実際に書き換える
python3 scripts/glossary-linker.py --apply
```

実行後、`git diff` で変更を確認してからコミット。

### どういう時に使うか

- **用語集に新語を追加したとき**：`/glossary/index.html` に `.term-card` を追加した後にこのスクリプトを流すと、ホワイトリスト記事で新語が自動的にリンク化される。
- **記事をリライトしたとき**：リライトで新しい専門用語が本文に入った場合、このスクリプトを流せばリンクが追加される。既存リンクは保護される。

### 設定の変更

スクリプト冒頭の以下を編集：

- `WHITELIST`：対象記事のリスト。新記事を対象に加える場合はここに追記。
- `EXCLUDED_SLUGS`：リンク化しない用語のスラッグ集合。文脈依存でリンクが不自然になる語を一律除外。

### 設計

- スラッグ↔表示語の対応表は `/glossary/index.html` の `.term-card` 構造から自動抽出
- 見出し・既存リンク・引用・JSON-LD・style等は除外ゾーンとして退避→復元
- 各記事で1つの用語は初出1回のみリンク化
- 記事内で既にリンク化済みの用語は、2箇所目以降もスキップ（過剰リンク化を防止）
- 差分40行を超える記事は警告を出す（中止はしない）
