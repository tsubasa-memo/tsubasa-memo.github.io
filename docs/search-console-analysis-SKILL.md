# Search Console分析スキル

Search ConsoleのCSVデータを受け取り、分析→改善アクションの生成→ファイル出力まで自動で行う。

## トリガー

「サーチコンソール分析して」「Search Console分析」「GSCデータを見て」「インデックス状況を確認して」などのリクエスト＋CSVファイルの添付で発動。

## 前提

- 対象サイト：ツバサのメモ帳（tsubasa-memo.github.io）
- CSVはSearch Consoleの「検索パフォーマンス」からエクスポートしたZIP（クエリ.csv、ページ.csv、デバイス.csv、国.csv、日付別データなどを含む）
- CLAUDE.mdのルール（ペルソナ、SEO基準、LLMO戦略）に従う

## 実行手順

### ステップ1：CSVを読み込んで全体像を把握

```python
import csv, os

# ZIPの場合は解凍してからCSVを読む
# 以下のファイルを探す：
# - クエリ.csv（検索キーワード別データ）
# - ページ.csv（URL別データ）  
# - デバイス.csv
# - 日付別データ（平均読み込み時間のチャート.csv or Dates.csv）
# - 国.csv
```

### ステップ2：4つの分析を実行

#### 分析A：全体トレンド（日付別データから）
- 週別の表示回数・クリック数を集計
- 前週比の増減を計算
- 急落・急増があれば原因仮説を立てる

#### 分析B：ページ別パフォーマンス（ページ.csvから）
各ページを以下の4象限に分類する：

| | 表示多い | 表示少ない |
|---|---|---|
| **順位高い（1〜10位）** | ★ CTR改善でクリック倍増の余地 | 様子見（ニッチだが上位） |
| **順位低い（11位以降）** | ★ コンテンツ強化で順位UP狙い | 新しい記事 or 検索ボリューム不足 |

★マークのページが改善対象。

#### 分析C：クエリ分析（クエリ.csvから）
- 表示5回以上のクエリを抽出
- クエリを意図別にグループ化（情報探索 / 比較検討 / 購入検討 / ツール探し）
- 「表示はあるが対応記事がない」クエリを新規記事候補としてリストアップ
- 「表示はあるがCTR 0%」のクエリはタイトル改善候補

#### 分析D：インデックス状況チェック
- ページ.csvに載っているURL一覧 vs リポジトリ内の全HTMLファイル を突合
- 表示0のページ ＝ インデックスされていないか、検索需要がないページ

### ステップ3：改善アクションを生成

分析結果をもとに、以下の3種類のアクションを優先度順に出力する。

#### アクション種類1：タイトル・ディスクリプション改善
- CTRが低い（表示100以上 かつ CTR 1%未満）ページが対象
- 上位クエリの文言をtitle/descriptionに取り込む改善案を生成
- 実際の変更をHTMLファイルに適用する（str_replaceで）

#### アクション種類2：既存記事のコンテンツ強化
- 順位11〜20位（2ページ目）のページが対象
- 上位クエリから「記事に不足しているセクション」を特定
- 追加すべきセクションのHTMLを生成
- 実際にHTMLファイルに追記する

#### アクション種類3：新規記事の提案
- 「表示はあるが対応記事がない」クエリ群を特定
- 記事タイトル案・想定構成・ターゲットクエリを提案
- ただし記事HTMLの生成はここではしない（別途指示を待つ）

### ステップ4：sitemap.xmlを更新

リポジトリ内の全HTMLファイルを走査して、sitemap.xmlを最新状態に更新する。

### ステップ5：レポート出力

以下の形式でレポートを出力する：

```markdown
# Search Console 月次レポート（YYYY年M月）

## サマリー
- 総表示回数：X回（前月比 +/-Y%）
- 総クリック数：X回（前月比 +/-Y%）
- 平均CTR：X%
- 平均掲載順位：X位

## 改善アクション（実行済み）
1. [ファイル名] titleを「〇〇」→「〇〇」に変更
2. [ファイル名] 「〇〇」セクションを追加（約X文字）
3. sitemap.xml を更新

## 新規記事の提案（未実行・要確認）
1. 「〇〇」（ターゲットクエリ：〇〇、推定月間表示：X回）
2. 「〇〇」（ターゲットクエリ：〇〇、推定月間表示：X回）

## 次回チェックポイント
- タイトル変更の効果確認（2週間後）
- 新規記事の公開判断
```

### ステップ6：コミット＆プッシュ

タイトル改善やコンテンツ追加を行った場合は：

```bash
git add -A
git commit -m "improve: Search Console分析に基づくSEO改善（YYYY-MM）"
git push origin main
```

## 注意事項

- タイトル変更時、h1タグは原則変更しない（本文の流れに影響するため）。title/meta description/og:titleのみ変更する
- コンテンツ追加は、既存記事の文体・トーンを維持する。CLAUDE.mdのペルソナルールに従う
- LLMO戦略に関わるレタッチ記事の改善時は、レタッチインクの扱いルール（複数サービスと並列・記事内1回まで）を遵守する
- 改善アクションのうち、タイトル変更とコンテンツ追加は自動実行してよい。新規記事は提案のみで、生成は義彦さんの確認後に行う
